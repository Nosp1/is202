Secure Webapp - Login Form

This is the login form page. You can design this page as you like, but there are some a couple of rules to follow:

  • The input fields for username and password must be named "j_username" and "j_password", respectively
  • and the value of the form's action attribute must be "j_security_check"

If the user attempts to go to a page that is protected by one of the security-constraints in web.xml, she will be redirected to the login page. On successful authentication she will be redirected again, this time to the page she was trying to get to. If the authentication fails she will be redirected to the failed login page. The names (or rather urls) of these pages are specified in web.xml.
