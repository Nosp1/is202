Installing netbeans and tomcat

The easiest way to install netbeans and tomcat, is to download the JavaEE bundle from netbean's download page (https://netbeans.org/downloads/. Run the installer. When asked to choose which servers to install, select tomcat, and deselect GlassFish and any other servers.

You will be asked for a username and password for a tomcat user. Enter something that you will remember.

Life is much easier if tomcat is the only web server you have installed, because any additional server may interfere with tomcat. For example other web servers may grab the ip ports that tomcat wants to use. So unless you are really good at configuring the different servers, you should not install any other than tomcat.

Things to remember

The installation creates a few files and directories (folders) that you will need later. They are:

CATALINA_HOME
This is the tomcat installation's main directory. The environment variable CATALINA_HOME will hold the path to this directory. Actually there isn't anything interesting in CATALINA_HOME. It's in the subdirectories.
CATALINA_HOME/conf/server.xml
This is the main configuration file for tomcat. If you make a mistake here, tomcat may stop working. So don't tamper with it. If you must, at least make a backup of the original file first
CATALINA_HOME/conf/Context.xml
This file serves the same purpose as the context.xml of the application, but the content here is valid for all applications, while the content of the project/application's context.xml is only valid for the application.
CATALINA_HOME/conf/tomcat-users.xml
This file contains tomcat's default user database. The admin user that was created during the installation of netbeans is stored here. You should be fine if you only add new users, but be careful to get the markup right.
